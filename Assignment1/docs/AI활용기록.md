## AI 활용기록

이번 과제를 진행하며 가장 크게 느꼈던 고민은 **'아 나는 평소 Java Spring 개발자인데 이렇게 디테일한 요구사항을 가진 FastAPI 서버를 기간내에 완벽하게(예외 처리, 안정성) 구현할 수 있을까?'**
였습니다.

하지만 이내 생각을 바꾸어 자신감을 얻게 된 계기는 최근 학습중이었던 **SDD(Spec-driven-development)** 때문이었습니다.

현재 언어에 대한 이해도는 부족하지만 백엔드 전반적인 개념은 결코 뒤쳐지지 않는다고 생각했습니다. 예를 들어 아래와 같은 개념들은 누구보다 자신있었습니다.
- OOP적 설계
- 예외 처리 구현 및 장애 대응
- 도커 활용 능력 등

그렇기에 인프라나 서버 설계 개념은 충분헀다고 판단하여 이번 과제의 중점은 `Speckit` + `Codex`를 활용한 기존의 지식을 FastAPI로 풀어내는 것에 중점을 맞췄습니다.

## 주요 작업 내용
제가 본질적으로 집중한것은 OOP 설계·예외 처리·장애 대응 같은 백엔드 본질이었습니다. 

**그렇다면 Codex와 Speckit은 어떠한 작업을 수행했을까요?**

- 기초 뼈대 FastAPI의 마이그레이션
- Docker Compose Gateway 관련 트러블슈팅
- 비즈니스 로직 작성 일부분


## 경험 요약
이번 과제를 한줄로 요약하자면, “AI가 거든 뼈대 + 제가 아는 운영 노하우” 조합은 생각 이상으로 효율적이었습니다.

비즈니스 로직에 대한 부분에선 평소의 저의 코드 스타일이나 중요 지점이라고 파악한 부분에서 예외 처리를 Codex에게 일임한는 것이 처음이었기에 사실은 신뢰도가 그렇게 높지않앗습니다.

그렇기에 기초 뼈대를 작성함과 동시에 Speckit을 활용하여 현재 프로젝트가 구성해야할 아키텍처, 코드 스타일, 예외처리, 트래픽 고려등의 부분을 일임하였고 이러한 생각은 효과적이었습니다.

만약 제가 메디솔브에이아이에서 업무를 맡아 새로운 기능 개발을 맡아야한다면 활용해볼 수 있을 것이라 생각되었습니다.
하지만 현재 레거시하게 구현되어있는 부분은 아직 `명세`만으로는 해결된다는 느낌이 들지않았고 사람의 손을 한번은 거쳐야한다는 것을 몸소 체험한 과제였습니다.
